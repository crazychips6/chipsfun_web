Chips Agent

通用 AI Agent 框架 — ReAct 循环 · 多工具编排 · 安全沙盒 · 持久记忆

概述

Chips Agent 是一个通用的 AI Agent 框架，采用 Python 3.12+ 构建。 它实现了一套完整的 ReAct（Reasoning + Acting）循环，将大语言模型与工具执行、持久记忆、 安全控制、沙盒环境、MCP 集成和可观测性连接在一起。

项目设计遵循"零依赖约束"——核心模块之间没有循环依赖，工具、安全、会话和记忆模块 对项目内部零依赖，保证了高度的模块化和可测试性。

ReAct 循环

核心循环 AIAgent.run_conversation() 最多执行 20 轮推理-行动迭代。 每一轮：构建 double-layer 系统提示（冻结缓存 + 动态感知）→ 调用 LLM → 处理 tool_calls → 分发到注册表 → 检测循环/故障 → 保存到会话数据库。 内置自动循环检测（4 次相同工具签名）和连续故障断路器（3 次失败）。

Prompt 构建

双阶段提示系统：build_frozen() 组装身份、记忆快照、技能索引， 一次构建整会话缓存（前缀缓存友好）；build_dynamic() 每轮重建 时间戳、预取上下文、工具集可用性表。内置提示注入检测（零宽字符、base64 编码指令、 Unicode 转义、中英文模式）。

核心特性

工具系统

15 个内置工具，覆盖终端、文件、Web、截图、进程、日历、地理位置等。 工具通过 registry.register() 自注册，支持运行时热启用/禁用工具集。

多工具编排

四种委派模式：single（单工具）、supervisor（顺序/并行）、pipeline（链式接力）、debate（辩论比较）。AgentPool 线程池并发控制。

上下文压缩

当提示超过 50% 上下文窗口时自动触发。修剪旧工具结果 → 保护头部/尾部 → LLM 摘要中间轮次。结构化摘要模板保留关键信息。

记忆系统

基于文件的 MEMORY.md/USER.md/EPISODIC.md 持久化，支持外部记忆提供者。 实验性全息记忆（HRR）引擎使用相位向量实现组合式记忆。

多层安全

三层命令审批：硬禁止（rm -rf / 等）→ 白名单 → 危险命令交互确认。 环境变量凭证剥离 + 文本模式清理。SHA-256 哈希链审计日志保证完整性。

沙盒环境

LocalEnvironment（子进程隔离 + 凭证剥离）和 DockerEnvironment（容器沙盒， --cap-drop ALL）。通过 Environment 协议抽象，可扩展自定义实现。

MCP 集成

通过 Model Context Protocol 连接外部工具服务器。自动工具注册命名空间避免冲突， 后台 asyncio 事件循环，配置文件在 ~/.chips/config.yaml。

可观测性

Prometheus 指标（LLM 调用/令牌/成本/延迟）、Langfuse 追踪（可选）、 结构化 JSON 行日志、会话级成本核算、按模型/工具的用量分析报表。

技能系统

基于 SKILL.md 文件的延迟加载技能（YAML frontmatter + markdown 正文）。 仅名称+描述在提示中，内容按需查看。灵感来自 Hermes 设计。

Web UI

FastAPI + SSE 流式后端，Next.js + TypeScript + Tailwind 前端。 JWT 认证，Prometheus /metrics 端点，Insights 分析 API。

编码约束

项目通过 CLAUDE.md 强制执行严格的分层依赖规则，保证架构整洁度：

  • tool/ — 零依赖，不引用任何其他内部模块
  • safety/ — 零依赖
  • session/ — 零依赖
  • memory/ — 零依赖
  • environment/ — 仅依赖 safety/
  • agent/ — 通过构造函数注入接收依赖，不直接 import 具体实现类

工具通过 registry.register() 自注册，Agent 只通过 registry.dispatch() 调用工具，不直接引用。
